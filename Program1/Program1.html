Entities and Attributes in a Portfolio Domain

- Domain: Portfolio -


Entity : PortfolioOwner

Sl. No. Attributes Data Type Justification
1 ownerId Integer

Unique identifier for the portfolio owner, auto-incremented
2 firstName String

Owner's first name
3 lastName String

Owner's last name
4 fullName String

Concatenated full name, derived or stored for convenience
5 tagLine String

Short descriptive phrase, text
6 email String

Contact email address, text with email format validation
7 gitHubUrl String

URL to GitHub profile, text with URL format validation
8 linkedInUrl String

URL to LinkedIn profile, text with URL format validation
9 freelancerUrl String

URL to Freelancer profile, text with URL format validation
10 profileImageUrl String

URL to profile image, if any, text with URL format validation


Entity : NavigationLink

Sl. No. Attributes Data Type Justification
1 linkId Integer

Unique identifier for the Navigation link, auto-incremented
2 linkText String

Text display for the link
3 targetSectionId String

Id of the HTML section the link point to
4 displayOrder Integer

The order in which the link appears in the navigation bar
5 isActive Boolean

Whether the link is currently active/visible
6 isExternal Boolean

Indicates if the link points to an external site
7 toolTipText String

Optional text for a tooltip on hover
8 iconClass String

Optional CSS class for an icon next to the link
9 creationDate Date

Date the link was added
10 lastModificationDate Date

Date the link was last modified


Entity : Section

Sl. No. Attributes Data Type Justification
1 sectionId String

Unique ID of the HTML section
2 sectionTitle String

Display's the title of the section
3 sectionDescription String

Brief introductory text for the section
4 displayOrder Integer

Order in which the section appears on the page
5 isVisible Boolean

Whether the section is currently visible/enabled
6 backgroundColor String

CSS color code for the section's background
7 paddingTop Integer

Padding at the top of the section
8 paddingBottom Integer

Padding at the bottom of the section
9 hasDivider Boolean

Indicdates if the section has a visual divider
10 creationDate Date

Date the section was defined


Entity : AboutMeDetail

Sl. No. Attributes Data Type Justification
1 detailId Integer

Unique identifier for the detail, auto-incremented
2 sectionId String

Foreign key referencing the "about" section
3 detailText String

The actual text of the detail
4 displayOrder Integer

Order in which the detail appears in the list
5 iconClass String

Optional CSS class for an icon next to the detail
6 isHighlight Boolean

Indicates if this is a key highlight
7 category String

e.g., "Skill", "Interest", "Personality Trait"
8 proficiencyLevel String

e.g., "Advanced", "Intermediate" for skills
9 startDate Date

When this detail became relevant, if applicable
10 endDate Date

When this detail ceased to be relevant, if applicable


Entity : Project

Sl. No. Attributes Data Type Justification
1 projectId Integer

Unique identifier for the project, auto-incremented
2 projectName String

Title of the project, e.g., "Project One"
3 projectDescription String

Detailed description of the project
4 projectUrl String

URL to the live project or repository, text with URL validation
5 thumbnailImageUrl String

URL to a thumbnail image for the project
6 technologiesUsed Array of Strings

List of technologies used, e.g., ["HTML", "CSS", "JS"]
7 completionDate Date

Date the project was completed
8 isFeatured Boolean

Whether the project is prominently featured
9 displayOrder Integer

Order in which projects are displayed
10 githubRepoUrl String

Direct link to the GitHub repository for the project


Entity : ContactMessage

Sl. No. Attributes Data Type Justification
1 messageId Integer

Unique identifier for the message, auto-incremented
2 senderName String

Name provided by the sender
3 senderEmail String

Email address provided by the sender, text with email format validation
4 messageContent Text

The actual message body
5 submissionTimestamp Date

Date and time the message was submitted
6 isRead Boolean

Whether the message has been read by the portfolio owner
7 isReplied Boolean

Whether a reply has been sent

IP address of the sender, for logging/security
9 userAgent String

10 sourcePage String

The page from which the message was sent, e.g., "/contact"
